## ТРЕБУЕТСЯ ОТ ТЕБЯ

### 1. Оценка бесплатных LLM API

Для каждого бесплатного API:
- **Лимиты:** requests/day, tokens/request
- **Качество NER/RE:** для русского языка + креативный домен
- **Достаточно ли** для обработки 10-20 seed URLs + периодическое обновление?
- **Стабильность:** не закроют ли free tier внезапно?

### 2. Новая архитектура для Budget = 0

Если бесплатные API существуют и подходят:
- Какой stack использовать для инфраструктуры (без AWS)?
- Python script + cron + бесплатный API + SQLite?
- Где хранить граф бесплатно?

Если бесплатных API недостаточно:
- Возможен ли **гибридный подход**: rule-based + minimal API calls?
- Или задача "автономный граф SOTA качества при budget = 0" **невыполнима**?

### 3. Trade-off анализ

**Realistic assessment:**
- Можно ли достичь confidence > 0.9 при budget = 0?
- Или нужно принять confidence ~0.7 как максимум?
- Стоит ли вообще строить систему, если качество будет низким?
